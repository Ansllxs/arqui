## Cómo realizar una comunicación con módulo?
Para esta comunicación, utilizaremos una pila, principalmente para casos de funciones dentro de otras funciones.

### Sistema FIFO (First In, First Out)
- Segmento de datos: para var globales
- Segmento de código: todo entre un **begin** y un **end;**
- Segmento de pila: segmento exlusivamente para procesos de pila

Cada fila de estos segmentos se le denomina **Stack Frame**

## Stack Frame genérico

En base a una función, tenemos:
- parametros
- valor a devolver
- lugar desde donde fue llamado

La manera más fácil de deshacerme de variables locales, es "popear" el stack frame de la pila, "matando" sus variables locales.

Por eficiencia, primero colocaremos el valor de retorno, quedandonos el siguiente stack frame, para funciones, en forma de pila:

- Variables locales
- Return address
- Parámetros
- Return value

El Stack Frame de un procedure es un poco distinto, pues no tiene ni variables locales ni return value.

- Tanto parámetros como return value son “creados” por el “caller module / módulo llamador”.
- Las variables locales son responsabilidad del “called module / módulo llamado”.
- El return address es “puesto” en la pila por el hardware.

El registro SS (Stack Segment) funciona como [indicador/puntero] hacia la cima del segmento de pila.

En ARM es responsabilidad de caller o del called, por ello
depende de la arquitectura.

## ¿Cómo se vería en ensamblador?

```
main:

    lea R8, a
    push R8
    lea R8, b
    push R8
    call getIntegers

Al hacer el procedure `getIntegers` debemos pasar la dirección de `a` y `b`.

Pila:

    RSP -> Ret. address
           *b
           *a

Además, hay un registro apuntando SIEMPRE al último elemento de la pila:

**RSP (Stack Pointer)**

Ya que las direcciones son de 64-bits / 8 bytes, entonces para asignar n y m, muevo RSP + [los bytes a mover]. Como Intel tiene una pila que crece hacia abajo, por eso se suma.
`m = RSP + 8`
`n = RSP + 16`

getIntegers:

    {I/O stuff?}
    {RAX <- n}
    mov RBX, [RSP+16]       ; obtengo la dirección
    mov [RBX], RAX          ; guardo en dicha dirección, el valor de RAX

    {more I/O stuff?}
    {RAX <- m}
    mov RBX, [RSP+8]
    mov [RBX], RAX
    ret                     ; toma aquello a donde apunte RSP, y lo pone sobre el registro RIP; si RSP no apunta a la dirección de retorno, entonces sucede el famoso pantallazo azul.

Al regresar a `call`, debo limpiar la pila,
para ello hay dos maneras.
```

- push y lea son instrucciones arquitectónicas para uso del segmento de pila
- call es una instrucción que pone el return address en la pila. Es un jump + colocación en pila.


## Las dos maneras de limpiar la pila:

1. Al ser procedure, no hay valor de retorno, no necesito nada más de ella. Para esta pila del ejemplo, simplemente:
- add RSP, 16
2. Ya que se le puede poner un constante al `ret`, entonces:
- ret 16 [en sí, hace lo mismo que add]
